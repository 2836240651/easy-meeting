## 快速测试步骤

### 1. 打开两个浏览器窗口

**窗口1（用户A - 主持人）：**
```
1. 打开 http://localhost:3000
2. 登录账号A
3. 创建会议
4. 点击"开启视频"按钮
5. 允许摄像头权限
6. 应该能看到自己的视频
```

**窗口2（用户B - 参与者）：**
```
1. 打开 http://localhost:3000（使用隐私模式或不同浏览器）
2. 登录账号B
3. 输入会议号加入会议
4. 点击"开启视频"按钮
5. 允许摄像头权限
6. 应该能看到自己的视频
```

### 2. 验证视频通话

**预期结果：**
- ✅ 用户A看到：自己的视频 + 用户B的视频
- ✅ 用户B看到：自己的视频 + 用户A的视频
- ✅ 视频实时传输，无明显延迟

### 3. 使用调试工具

在浏览器控制台（F12）输入：
```javascript
// 查看WebRTC连接状态
debugWebRTC()

// 查看所有peer连接
webrtcManager.peerConnections

// 查看远程视频流
webrtcManager.remoteStreams
```

## 常见问题

### Q1: 看不到对方的视频，只能看到自己的

**检查：**
1. 打开浏览器控制台，输入 `debugWebRTC()`
2. 查看 `Peer连接数量` 是否为1
3. 查看 `远程流数量` 是否为1
4. 查看连接状态是否为 `connected`

**如果连接数量为0：**
- 检查WebSocket是否连接：`meetingWsService.isConnected`
- 查看后端日志是否有 "发送点对点消息"
- 检查是否有JavaScript错误

### Q2: 视频显示黑屏

**检查：**
1. 摄像头是否被其他应用占用
2. 浏览器是否授予摄像头权限
3. 控制台是否有错误信息

### Q3: 连接状态一直是connecting

**可能原因：**
- ICE候选交换失败
- STUN服务器不可达
- 防火墙阻止

**解决：**
1. 检查网络连接
2. 尝试关闭防火墙测试
3. 查看控制台ICE相关日志

## 性能提示

- **2-4人会议**：流畅运行
- **5-6人会议**：可能有轻微卡顿
- **7人以上**：不推荐使用Mesh架构

如果需要支持更多人，建议使用SFU架构（需要媒体服务器）。
