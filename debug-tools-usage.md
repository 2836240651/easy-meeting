# 调试工具使用说明

## 工具列表

### 1. quick-participants-test.html（推荐）
**用途**：快速测试会议中有多少成员

**特点**：
- ✅ 内置登录功能
- ✅ 界面简洁友好
- ✅ 一键测试
- ✅ 结果清晰明了

**使用步骤**：
1. 直接在浏览器中打开 `quick-participants-test.html`
2. 输入邮箱和密码登录
3. 点击"开始测试"按钮
4. 查看测试结果

**测试结果说明**：
- 显示大数字（如 0、1、2）表示会议中其他成员的数量
- 绿色框：测试成功
- 黄色框：警告（如会议中只有你一个人）
- 红色框：错误（如未登录、未加入会议）

---

### 2. debug-participants-display-v2.html
**用途**：详细查看API返回的原始数据

**特点**：
- ✅ 内置登录功能
- ✅ 显示完整JSON数据
- ✅ 分步骤展示调试信息
- ✅ 可以看到每个成员的详细字段

**使用步骤**：
1. 直接在浏览器中打开 `debug-participants-display-v2.html`
2. 输入邮箱和密码登录
3. 点击"刷新会议信息"查看当前会议
4. 点击"加载成员列表"查看所有成员
5. 查看5个调试区域的详细信息

**调试区域说明**：
1. **用户信息检查**：显示当前登录用户的信息
2. **会议信息检查**：显示当前会议的详情
3. **会议成员列表（API返回）**：显示后端返回的原始数据
4. **过滤后的成员列表**：显示前端过滤后的数据
5. **成员头像预览**：可视化显示每个成员的头像

---

## 常见使用场景

### 场景1：检查会议中有多少人
**推荐工具**：quick-participants-test.html

**步骤**：
1. 打开工具并登录
2. 点击"开始测试"
3. 查看显示的数字

**结果判断**：
- 显示 `0`：会议中只有你一个人（正常）
- 显示 `1+`：会议中有其他成员

---

### 场景2：查看成员的详细信息
**推荐工具**：debug-participants-display-v2.html

**步骤**：
1. 打开工具并登录
2. 点击"加载成员列表"
3. 查看"会议成员列表（API返回）"区域的JSON数据

**可以看到的信息**：
- userId：用户ID
- nickName：昵称
- avatar：头像URL
- sex：性别（0=女，1=男）
- status：状态（1=正常，2=已退出）
- memberType：类型（0=普通成员，1=主持人）

---

### 场景3：测试多人会议
**推荐工具**：quick-participants-test.html

**步骤**：
1. **浏览器1**（用户A）：
   - 打开工具并登录账号A
   - 创建快速会议（在主应用中）
   - 记录会议号
   - 打开测试工具，点击"开始测试"
   - 应该显示 `0`（只有自己）

2. **浏览器2**（用户B）：
   - 打开另一个浏览器或隐身窗口
   - 打开工具并登录账号B
   - 加入用户A的会议号（在主应用中）

3. **回到浏览器1**：
   - 刷新测试工具页面
   - 重新登录（如果需要）
   - 点击"开始测试"
   - 应该显示 `1`（有1个其他成员）

## 登录说明

### 为什么需要登录？
调试工具需要访问后端API来获取会议和成员信息，这些API需要用户token进行身份验证。

### 登录信息存储
登录后，token和用户信息会保存在浏览器的localStorage中，下次打开工具时会自动登录。

### 退出登录
点击"退出登录"按钮会清除localStorage中的登录信息。

### 多账号测试
如果需要测试多个账号：
1. 使用不同的浏览器（Chrome、Firefox、Edge等）
2. 或使用隐身/无痕模式
3. 每个浏览器/窗口可以登录不同的账号

## 故障排除

### 问题1：登录失败
**可能原因**：
- 邮箱或密码错误
- 后端服务未启动
- 网络连接问题

**解决方法**：
1. 检查邮箱和密码是否正确
2. 确认后端服务运行在 http://localhost:6099
3. 打开浏览器控制台（F12）查看错误信息

---

### 问题2：显示"未找到当前会议"
**可能原因**：
- 用户未创建或加入会议
- 会议已结束

**解决方法**：
1. 先在主应用中创建或加入一个会议
2. 然后再使用调试工具测试

---

### 问题3：显示"会议中没有其他成员"
**这不是问题！**

这是正常情况，说明：
- 会议中只有你一个人
- 需要另一个用户加入会议才能看到其他成员

**测试方法**：
1. 打开另一个浏览器
2. 用另一个账号登录
3. 加入你的会议号
4. 回到第一个浏览器刷新测试

---

### 问题4：CORS跨域错误
**可能原因**：
- 直接用 `file://` 协议打开HTML文件
- 浏览器安全策略限制

**解决方法**：
1. 使用本地服务器打开（如 Live Server）
2. 或者在主应用中打开调试工具
3. 或者配置后端允许跨域

---

## 技术说明

### API端点
- 登录：`POST /api/account/login`
- 获取当前会议：`GET /api/meetingInfo/getCurrentMeeting`
- 获取成员列表：`GET /api/meetingInfo/loadMeetingMembers`

### 成员状态枚举
```javascript
MemberStatus = {
    DEL_MEETING: 0,   // 删除会议
    NORMAL: 1,        // 正常（在会议中）
    EXIT_MEETING: 2,  // 退出会议
    KICK_OUT: 3,      // 被踢出
    BLACKLIST: 4      // 被拉黑
}
```

### 过滤逻辑
前端只显示满足以下条件的成员：
1. `status === 1` (NORMAL)
2. `userId !== currentUserId` (不是当前用户)
