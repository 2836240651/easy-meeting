# 音频独立控制功能

## 功能说明

音频和视频现在完全独立控制，支持以下场景：

### 场景1: 纯音频会议（不开视频）
1. 进入会议
2. 不点击视频按钮
3. 直接点击"解除静音"按钮
4. 系统会自动获取麦克风权限
5. 可以进行纯音频通话

### 场景2: 先音频后视频
1. 进入会议
2. 点击"解除静音"→ 获取音频
3. 稍后点击"视频"→ 添加视频
4. 音频和视频同时工作

### 场景3: 先视频后音频
1. 进入会议
2. 点击"视频"→ 获取视频和音频
3. 点击"静音"→ 关闭音频
4. 点击"解除静音"→ 恢复音频

### 场景4: 只开视频不开音频
1. 进入会议
2. 点击"视频"→ 获取视频和音频
3. 点击"静音"→ 关闭音频
4. 保持视频开启，音频静音

## 技术实现

### 独立获取音频流
```javascript
const getAudioStream = async () => {
  const audioStream = await navigator.mediaDevices.getUserMedia({
    video: false,  // 只获取音频
    audio: {
      echoCancellation: true,
      noiseSuppression: true,
      autoGainControl: true
    }
  })
  
  // 如果已有视频流，添加音频轨道
  if (localStream.value) {
    const audioTrack = audioStream.getAudioTracks()[0]
    localStream.value.addTrack(audioTrack)
  } else {
    // 创建新的纯音频流
    localStream.value = audioStream
  }
}
```

### 智能切换逻辑
```javascript
const toggleMute = async () => {
  // 如果要解除静音但没有流，获取音频
  if (!isMuted.value && !localStream.value) {
    await getAudioStream()
    return
  }
  
  // 如果有流但没有音频轨道，添加音频
  if (localStream.value && audioTracks.length === 0) {
    await getAudioStream()
    return
  }
  
  // 如果有音频轨道，切换enabled状态
  audioTracks.forEach(track => {
    track.enabled = !isMuted.value
  })
}
```

## 优势

### 1. 灵活性
- 用户可以选择只用音频（节省带宽）
- 可以随时添加/移除视频
- 独立控制音频和视频

### 2. 用户体验
- 不强制开启视频
- 适合不同场景（会议、语音聊天等）
- 权限请求更友好

### 3. 性能优化
- 纯音频模式占用更少带宽
- 适合网络较差的环境
- 降低CPU使用率

## 常见问题

### Q: 点击解除静音没反应？
A: 检查：
1. 浏览器是否允许麦克风权限
2. 系统是否有可用的麦克风设备
3. 浏览器控制台是否有错误

### Q: 添加音频后视频中断？
A: 不应该发生，如果发生：
1. 检查浏览器控制台错误
2. 确认 `addTrack` 方法是否成功
3. 检查WebRTC连接状态

### Q: 音频质量不好？
A: 
- 检查麦克风质量
- 检查网络连接
- 可以调整音频约束参数
