# 默认视频关闭功能

## 修改内容

用户进入会议时默认不开启视频，需要手动点击"开启视频"按钮才能开启摄像头。

## 修改的文件

### frontend/src/views/Meeting.vue

#### 1. 会议状态初始值
```javascript
// 修改前
const isVideoOn = ref(true)

// 修改后
const isVideoOn = ref(false)  // 默认不开启视频
```

#### 2. 设置默认值
```javascript
// 修改前
const settings = ref({
  autoMute: false,
  autoVideo: true,
  showNotifications: true
})

// 修改后
const settings = ref({
  autoMute: false,
  autoVideo: false,  // 默认不自动开启视频
  showNotifications: true
})
```

## 用户体验

### 进入会议时
- ✅ 视频默认关闭，显示用户头像
- ✅ 视频按钮显示"开启视频"
- ✅ 视频按钮有 `active` 样式（高亮）
- ✅ 参与者状态显示"📹"图标（视频关闭）

### 开启视频
- 用户点击"开启视频"按钮
- 浏览器请求摄像头权限
- 如果有多个摄像头，显示选择界面
- 视频开启后，按钮文本变为"关闭视频"

## 优势

1. **隐私保护** - 用户可以先确认环境再开启视频
2. **带宽节省** - 不需要视频的会议可以节省带宽
3. **用户控制** - 给用户更多控制权
4. **符合习惯** - 大多数会议软件都是默认关闭视频

## 相关功能

- 音频默认也是关闭的（`isMuted = false` 表示未静音，但没有音频流）
- 用户可以在设置中修改 `autoVideo` 选项（未来可以保存到用户偏好）
- WebRTC 连接仍然会建立，只是不发送视频轨道

## 测试验证

1. 清除浏览器缓存
2. 进入会议
3. 确认：
   - 显示头像而不是视频
   - 按钮显示"开启视频"
   - 没有摄像头权限请求
4. 点击"开启视频"
5. 确认：
   - 浏览器请求摄像头权限
   - 视频正常显示
   - 其他用户能看到视频
