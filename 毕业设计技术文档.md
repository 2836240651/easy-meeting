# 基于SpringBoot的多人在线会议系统 - 毕业设计技术文档

## 一、项目概述

### 1.1 项目背景
随着远程办公和在线教育的普及,视频会议系统已成为现代社会不可或缺的通信工具。本项目基于SpringBoot框架开发了一套完整的多人在线会议系统,实现了实时音视频通信、屏幕共享、会议管理等核心功能。

### 1.2 项目目标
- 实现稳定可靠的多人实时音视频通信
- 提供完善的会议管理功能(创建、预约、加入会议)
- 支持屏幕共享、实时聊天等协作功能
- 实现用户管理、好友系统、通知系统等辅助功能
- 提供跨平台桌面应用支持(基于Electron)

### 1.3 技术栈

## 三、技术重点

### 3.1 WebRTC实时音视频通信 ⭐⭐⭐⭐⭐

**技术原理:**
WebRTC (Web Real-Time Communication) 是一个支持网页浏览器进行实时语音对话或视频对话的API。本系统采用Mesh架构实现点对点(P2P)连接。

**实现方案:**
1. **信令服务器**: 使用WebSocket作为信令通道,交换SDP和ICE候选
2. **P2P连接**: 每个参与者与其他所有参与者建立直接连接
3. **媒体流管理**: 支持音频、视频轨道的独立控制
4. **NAT穿透**: 使用STUN/TURN服务器实现NAT穿透

**核心代码结构:**
```javascript
// WebRTC管理器 (frontend/src/api/webrtc-manager.js)
class WebRTCManager {
  constructor() {
    this.peerConnections = new Map()  // 存储所有P2P连接
    this.remoteStreams = new Map()     // 存储远程媒体流
    this.localStream = null            // 本地媒体流
  }
  
  // 创建P2P连接
  createPeerConnection(targetUserId) {
    const pc = new RTCPeerConnection({
      iceServers: [
        { urls: 'stun:stun.l.google.com:19302' }
      ]
    })
    
    // 添加本地流
    this.localStream.getTracks().forEach(track => {
      pc.addTrack(track, this.localStream)
    })
    
    // 监听ICE候选
    pc.onicecandidate = (event) => {
      if (event.candidate) {
        this.sendIceCandidate(targetUserId, event.candidate)
      }
    }
    
    // 监听远程流
    pc.ontrack = (event) => {
      this.handleRemoteTrack(targetUserId, event.streams[0])
    }
    
    return pc
  }
  
  // 创建Offer
  async createOffer(targetUserId) {
    const pc = this.createPeerConnection(targetUserId)
    const offer = await pc.createOffer()
    await pc.setLocalDescription(offer)
    this.sendOffer(targetUserId, offer)
  }
  
  // 处理Answer
  async handleAnswer(fromUserId, answer) {
    const pc = this.peerConnections.get(fromUserId)
    await pc.setRemoteDescription(new RTCSessionDescription(answer))
  }
}
```

**技术亮点:**
1. 实现了完整的WebRTC信令流程(Offer/Answer/ICE)
2. 支持动态添加/移除参与者
3. 实现了音视频轨道的独立控制(可单独开关摄像头/麦克风)
4. 处理了网络断线重连场景
5. 支持多种视频质量设置(360p/480p/720p/1080p)


### 3.2 WebSocket双向通信机制 ⭐⭐⭐⭐⭐

**技术原理:**
WebSocket提供了全双工通信通道,实现服务器主动推送消息给客户端,是实现实时通信的关键技术。

**实现方案:**
1. **后端**: 使用t-io框架实现高性能WebSocket服务器
2. **心跳机制**: 客户端每30秒发送心跳,服务器响应保持连接
3. **消息路由**: 根据消息类型分发到不同的处理器
4. **断线重连**: 客户端检测到断线后自动重连

**后端核心实现:**
```java
// WebSocket处理器
@Component
public class MessageHandler {
    
    @Autowired
    private RedisComponent redisComponent;
    
    // 处理不同类型的消息
    public void handleMessage(ChannelContext channelContext, WsMessage message) {
        MessageTypeEnum messageType = MessageTypeEnum.getByType(message.getMessageType());
        
        switch (messageType) {
            case HEART_BEAT:
                handleHeartBeat(channelContext);
                break;
            case MEMBER_JOINED:
                handleMemberJoined(channelContext, message);
                break;
            case WEBRTC_OFFER:
                handleWebRTCOffer(channelContext, message);
                break;
            case WEBRTC_ANSWER:
                handleWebRTCAnswer(channelContext, message);
                break;
            case ICE_CANDIDATE:
                handleIceCandidate(channelContext, message);
                break;
            case CHAT_MESSAGE:
                handleChatMessage(channelContext, message);
                break;
        }
    }
    
    // 广播消息到会议室所有成员
    public void broadcastToMeeting(String meetingId, WsMessage message, String excludeUserId) {
        Set<ChannelContext> channels = ChannelContextUtils.getMeetingChannels(meetingId);
        for (ChannelContext ctx : channels) {
            String userId = (String) ctx.get("userId");
            if (!userId.equals(excludeUserId)) {
                Tio.send(ctx, message);
            }
        }
    }
    
    // 发送消息给指定用户
    public void sendToUser(String userId, WsMessage message) {
        ChannelContext ctx = ChannelContextUtils.getUserChannel(userId);
        if (ctx != null) {
            Tio.send(ctx, message);
        }
    }
}
```

**前端核心实现:**
```javascript
// WebSocket服务
class WebSocketService {
  constructor() {
    this.ws = null
    this.heartbeatTimer = null
    this.reconnectTimer = null
    this.messageHandlers = new Map()
  }
  
  connect(token, userId) {
    this.ws = new WebSocket(`ws://localhost:8081/ws?token=${token}`)
    
    this.ws.onopen = () => {
      console.log('WebSocket连接成功')
      this.startHeartbeat()
    }
    
    this.ws.onmessage = (event) => {
      const message = JSON.parse(event.data)
      this.routeMessage(message)
    }
    
    this.ws.onclose = () => {
      console.log('WebSocket连接关闭')
      this.stopHeartbeat()
      if (this.shouldReconnect) {
        this.reconnect()
      }
    }
  }
  
  // 心跳机制
  startHeartbeat() {
    this.heartbeatTimer = setInterval(() => {
      this.send({
        messageType: 1,  // HEART_BEAT
        messageContent: 'ping'
      })
    }, 30000)
  }
  
  // 消息路由
  routeMessage(message) {
    const handlers = this.messageHandlers.get(message.messageType)
    if (handlers) {
      handlers.forEach(handler => handler(message))
    }
  }
  
  // 注册消息处理器
  on(messageType, handler) {
    if (!this.messageHandlers.has(messageType)) {
      this.messageHandlers.set(messageType, [])
    }
    this.messageHandlers.get(messageType).push(handler)
  }
}
```

**技术亮点:**
1. 实现了可靠的心跳保活机制,防止连接超时
2. 支持消息类型的灵活路由,易于扩展
3. 实现了智能断线重连,提高系统可用性
4. 使用Redis管理WebSocket连接状态,支持分布式部署
5. 实现了消息的精准推送(单播、广播、组播)


### 3.3 屏幕共享功能 ⭐⭐⭐⭐

**技术原理:**
使用WebRTC的getDisplayMedia API获取屏幕流,通过P2P连接传输给其他参与者。

**实现方案:**
1. **屏幕捕获**: 使用navigator.mediaDevices.getDisplayMedia()
2. **流替换**: 将摄像头流替换为屏幕流
3. **画中画**: 在共享屏幕时显示共享者的摄像头画面
4. **观看者视图**: 为观看者提供专门的屏幕共享观看界面

**核心代码:**
```javascript
// 开始屏幕共享
async shareScreen() {
  try {
    // 获取屏幕流
    const screenStream = await navigator.mediaDevices.getDisplayMedia({
      video: {
        cursor: 'always',
        displaySurface: 'monitor',
        frameRate: { ideal: 30 },
        width: { ideal: 1920 },
        height: { ideal: 1080 }
      },
      audio: this.screenShareOptions.shareAudio
    })
    
    // 保存原始摄像头流
    this.originalCameraStream = this.localStream
    
    // 替换视频轨道
    const videoTrack = screenStream.getVideoTracks()[0]
    const sender = this.peerConnection.getSenders()
      .find(s => s.track.kind === 'video')
    await sender.replaceTrack(videoTrack)
    
    // 监听屏幕共享停止
    videoTrack.onended = () => {
      this.stopScreenShare()
    }
    
    // 通知其他参与者
    this.notifyScreenShareStart()
    
  } catch (error) {
    console.error('屏幕共享失败:', error)
  }
}

// 停止屏幕共享
async stopScreenShare() {
  // 恢复摄像头流
  const videoTrack = this.originalCameraStream.getVideoTracks()[0]
  const sender = this.peerConnection.getSenders()
    .find(s => s.track.kind === 'video')
  await sender.replaceTrack(videoTrack)
  
  // 通知其他参与者
  this.notifyScreenShareStop()
}
```

**技术亮点:**
1. 支持共享整个屏幕、应用窗口或浏览器标签页
2. 实现了画中画功能,共享者可以看到自己的摄像头
3. 观看者有专门的全屏观看模式
4. 支持共享系统音频
5. 实现了屏幕共享的优雅停止和恢复


### 3.4 会议管理系统 ⭐⭐⭐⭐

**功能模块:**
1. **快速会议**: 立即创建并开始会议
2. **预约会议**: 预约未来时间的会议
3. **加入会议**: 通过会议号加入会议
4. **会议控制**: 主持人可以踢出成员、结束会议

**核心实现:**
```java
// 会议服务实现
@Service
public class MeetingInfoServiceImpl implements MeetingInfoService {
    
    // 创建会议
    @Override
    @Transactional
    public String createMeeting(String userId, String meetingName, Boolean videoOpen) {
        // 1. 生成唯一会议号
        String meetingNo = generateMeetingNo();
        
        // 2. 创建会议记录
        MeetingInfo meeting = new MeetingInfo();
        meeting.setMeetingNo(meetingNo);
        meeting.setMeetingName(meetingName);
        meeting.setHostUserId(userId);
        meeting.setStatus(0); // 进行中
        meeting.setCreateTime(new Date());
        this.save(meeting);
        
        // 3. 添加主持人为会议成员
        MeetingMember member = new MeetingMember();
        member.setMeetingId(meeting.getMeetingId());
        member.setUserId(userId);
        member.setIsHost(true);
        member.setVideoOpen(videoOpen);
        member.setStatus(0);
        meetingMemberService.save(member);
        
        // 4. 缓存会议信息到Redis
        redisComponent.saveMeetingInfo(meeting);
        
        return meeting.getMeetingId();
    }
    
    // 加入会议
    @Override
    @Transactional
    public void joinMeeting(String userId, String meetingId, Boolean videoOpen) {
        // 1. 验证会议是否存在且进行中
        MeetingInfo meeting = this.getById(meetingId);
        if (meeting == null || meeting.getStatus() != 0) {
            throw new BusinessException("会议不存在或已结束");
        }
        
        // 2. 检查是否已在会议中
        MeetingMember existMember = meetingMemberService.getMember(meetingId, userId);
        if (existMember != null && existMember.getStatus() == 0) {
            return; // 已在会议中
        }
        
        // 3. 添加为会议成员
        MeetingMember member = new MeetingMember();
        member.setMeetingId(meetingId);
        member.setUserId(userId);
        member.setIsHost(false);
        member.setVideoOpen(videoOpen);
        member.setStatus(0);
        member.setJoinTime(new Date());
        meetingMemberService.save(member);
        
        // 4. 通知其他成员
        notifyMemberJoined(meetingId, userId);
    }
    
    // 结束会议
    @Override
    @Transactional
    public void finishMeeting(String meetingId, String userId) {
        MeetingInfo meeting = this.getById(meetingId);
        
        // 验证是否为主持人
        if (!meeting.getHostUserId().equals(userId)) {
            throw new BusinessException("只有主持人可以结束会议");
        }
        
        // 更新会议状态
        meeting.setStatus(1); // 已结束
        meeting.setEndTime(new Date());
        this.updateById(meeting);
        
        // 更新所有成员状态
        meetingMemberService.finishAllMembers(meetingId);
        
        // 通知所有成员会议已结束
        notifyMeetingFinished(meetingId);
        
        // 清理Redis缓存
        redisComponent.deleteMeetingInfo(meetingId);
    }
}
```

**技术亮点:**
1. 使用事务保证数据一致性
2. 使用Redis缓存提高查询性能
3. 实现了会议状态的完整生命周期管理
4. 支持断线重连后恢复会议状态
5. 实现了主持人权限控制

// 会议预约服务
@Service
public class MeetingReserveServiceImpl implements MeetingReserveService {
    
    // 创建预约
    @Override
    @Transactional
    public String createReservation(MeetingReserveDto dto) {
        // 1. 验证预约时间
        if (dto.getStartTime().before(new Date())) {
            throw new BusinessException("预约时间不能早于当前时间");
        }
        
        // 2. 生成会议号
        String meetingNo = generateMeetingNo();
        
        // 3. 创建预约记录
        MeetingReserve reserve = new MeetingReserve();
        reserve.setMeetingNo(meetingNo);
        reserve.setMeetingName(dto.getMeetingName());
        reserve.setHostUserId(dto.getHostUserId());
        reserve.setStartTime(dto.getStartTime());
        reserve.setDuration(dto.getDuration());
        reserve.setPassword(dto.getPassword());
        reserve.setStatus(0); // 待开始
        this.save(reserve);
        
        // 4. 发送邀请通知
        if (dto.getInviteUserIds() != null) {
            sendInviteNotifications(reserve, dto.getInviteUserIds());
        }
        
        // 5. 设置提醒任务
        scheduleReminder(reserve);
        
        return reserve.getReserveId();
    }
    
    // 发送会议提醒
    private void scheduleReminder(MeetingReserve reserve) {
        // 获取用户设置的提醒时间(默认10分钟)
        int reminderMinutes = userSettingsService.getReminderTime(reserve.getHostUserId());
        
        // 计算提醒时间
        Date reminderTime = new Date(reserve.getStartTime().getTime() - reminderMinutes * 60 * 1000);
        
        // 创建定时任务
        taskScheduler.schedule(() -> {
            sendMeetingReminder(reserve);
        }, reminderTime);
    }
    
    // 发送会议提醒通知
    private void sendMeetingReminder(MeetingReserve reserve) {
        UserNotification notification = new UserNotification();
        notification.setUserId(reserve.getHostUserId());
        notification.setNotificationType(6); // 会议提醒
        notification.setTitle("会议即将开始");
        notification.setContent(String.format("您预约的会议「%s」将在%d分钟后开始", 
            reserve.getMeetingName(), reminderMinutes));
        notification.setRelatedId(reserve.getReserveId());
        notificationService.save(notification);
        
        // 通过WebSocket推送实时通知
        wsService.sendToUser(reserve.getHostUserId(), notification);
    }
}

    @Override

        wrapper.eq("user_id", query.getUserId());

@Component
public class RedisLockService {
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    // 获取锁
    public boolean tryLock(String key, String value, long timeout) {
        Boolean result = redisTemplate.opsForValue()
            .setIfAbsent(key, value, timeout, TimeUnit.SECONDS);
        return Boolean.TRUE.equals(result);
    }
    
    // 释放锁
    public void unlock(String key, String value) {
        String currentValue = redisTemplate.opsForValue().get(key);
        if (value.equals(currentValue)) {
            redisTemplate.delete(key);
        }
    }
}

### 7.1 身份认证安全

**1. JWT Token机制**
```java
public class JwtUtil {
    private static final String SECRET_KEY = "your-secret-key";
    private static final long EXPIRATION_TIME = 7 * 24 * 60 * 60 * 1000; // 7天
    
    // 生成Token
    public static String generateToken(String userId) {
        return Jwts.builder()
            .setSubject(userId)
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + EXPIRATION_TIME))
            .signWith(SignatureAlgorithm.HS512, SECRET_KEY)
            .compact();
    }
    
    // 验证Token
    public static String validateToken(String token) {
        try {
            Claims claims = Jwts.parser()
                .setSigningKey(SECRET_KEY)
                .parseClaimsJws(token)
                .getBody();
            return claims.getSubject();
        } catch (Exception e) {
            throw new BusinessException("Token无效");
        }
    }
}
```
