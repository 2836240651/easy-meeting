🔍 调试参与者显示问题

问题描述

会议页面只显示自己的头像，不显示其他成员的头像。

可能的原因

  1. participants 数组为空 - 没有加载到其他成员数据
  2. 过滤逻辑错误 - 把所有成员都过滤掉了
  3. currentUserId 未设置 - 导致无法正确过滤
  4. API返回数据问题 - 后端只返回了当前用户
  5. WebSocket未连接 - 没有收到其他成员加入的消息

调试步骤

  • 打开浏览器开发者工具（F12）
  • 切换到 Console 标签页
  • 加入或创建一个会议
  • 查看控制台输出的日志
  • 检查以下关键信息

关键日志检查

1. 检查 currentUserId

在 Console 中输入： localStorage.getItem('userInfo') 应该看到包含 userId 的 JSON 数据

2. 检查 participants 数组

查找控制台日志： "更新参与者列表，当前人数（不包括自己）: X" 如果 X = 0，说明没有其他成员 如果 X > 0，说明有其他成员但没有显示

3. 检查 API 响应

在 Network 标签页中查找： loadMeetingMembers 查看响应数据中有多少个成员

4. 检查过滤逻辑

查找控制台日志： "成员状态检查: XXX status: X userId: XXX currentUserId: XXX" 检查： - status 是否等于 1 (MemberStatus.NORMAL) - userId 是否与 currentUserId 不同

解决方案

问题 解决方案
participants 数组为空 1. 确保调用了 loadParticipants()
2. 检查 API 是否返回数据
3. 检查 meetingId 是否正确
过滤逻辑把所有成员都过滤掉了 1. 检查 member.status 的值
2. 检查 MemberStatus.NORMAL 的值
3. 确保 currentUserId 已设置

2. 检查 WebSocket 是否连接
3. 查看是否收到 memberJoined 消息

2. 确保登录成功
3. 刷新页面重新登录

代码检查清单

Meeting.vue 中需要检查的地方：

1. updateParticipantsList 函数：
   - 是否正确过滤：member.status === MemberStatus.NORMAL && member.userId !== currentUserId.value
   - 是否正确映射：participants.value = filteredMembers.map(...)

2. loadParticipants 函数：
   - 是否被调用
   - meetingId 是否有值
   - API 响应是否正确

3. joinMeeting 函数：
   - 是否调用了 loadParticipants()
   - 调用顺序是否正确

4. HTML 模板：
   - v-for="participant in participants" 是否正确
   - 是否排除了当前用户（已单独显示）

5. currentUserId：
   - 是否在 loadUserInfo() 中正确设置
   - 值是否与后端返回的 userId 一致

快速修复建议

已修改的代码： updateParticipantsList 函数现在会： 1. 过滤状态为 NORMAL 的成员 2. 排除当前用户（避免重复显示） 3. 格式化成员数据 4. 更新 participants 数组 如果还是只显示自己： 1. 打开浏览器控制台 2. 查找日志："过滤后成员数量（排除自己）: X" 3. 如果 X = 0： - 检查是否有其他人加入会议 - 检查 API 返回的数据 4. 如果 X > 0 但不显示： - 检查 HTML 模板中的 v-for - 检查 CSS 样式是否隐藏了元素
