# 屏幕共享 UI 功能测试指南

## 问题解决

### Vite 500 错误
如果遇到 `GET http://localhost:3000/src/views/Meeting.vue net::ERR_ABORTED 500` 错误：

**原因：** Vite 开发服务器在处理大文件或热更新时可能出现问题

**解决方案：** 重启 Vite 开发服务器
```bash
cd frontend
# 停止当前服务器 (Ctrl+C)
npm run dev
```

## 测试步骤

### 1. 启动应用
```bash
# 后端服务器（如果未运行）
mvn spring-boot:run

# 前端开发服务器（已重启）
cd frontend
npm run dev

# Electron 应用
npm run electron:dev
```

### 2. 测试屏幕共享选项对话框

1. 登录应用
2. 创建或加入会议
3. 点击底部控制栏的"共享屏幕"按钮
4. **验证：** 应该弹出屏幕共享选项对话框
5. **验证：** 对话框包含两个复选框：
   - ☑️ 同时共享电脑声音
   - ☑️ 人像画中画（默认勾选）
6. **验证：** 对话框包含"取消"和"开始共享"按钮
7. 勾选/取消勾选选项，验证状态变化
8. 点击"取消"，验证对话框关闭

### 3. 测试屏幕共享工具栏

1. 在选项对话框中点击"开始共享"
2. 选择要共享的屏幕/窗口
3. **验证：** 屏幕共享开始
4. **验证：** 顶部出现工具栏（可能需要将鼠标移到屏幕顶部）
5. **验证：** 工具栏包含以下内容：

   **左侧：**
   - 会议详情按钮
   - 会议时长显示（格式：MM:SS 或 HH:MM:SS）
   - 分享会议按钮

   **中间：**
   - 静音/解除静音按钮
   - 开启/关闭视频按钮
   - 成员按钮
   - 聊天按钮
   - 录制按钮
   - 暂停共享按钮
   - 结束共享按钮

### 4. 测试工具栏自动隐藏

1. 开始屏幕共享后，工具栏应该显示
2. 将鼠标移开屏幕顶部
3. **验证：** 3秒后工具栏自动向上滑动隐藏
4. 将鼠标移到屏幕顶部
5. **验证：** 工具栏自动向下滑动显示
6. 重复几次验证流畅性

### 5. 测试会议时长计时器

1. 观察工具栏左侧的会议时长显示
2. **验证：** 时长每秒更新一次
3. **验证：** 格式正确：
   - 小于1小时：MM:SS（如 05:30）
   - 大于1小时：HH:MM:SS（如 01:05:30）
4. **验证：** 时间持续增加

### 6. 测试工具栏功能按钮

#### 会议详情
1. 点击"会议详情"按钮
2. **验证：** 弹出会议信息浮框
3. **验证：** 显示会议名称、会议号、主持人等信息

#### 分享会议
1. 点击"分享会议"按钮
2. **验证：** 显示提示"会议信息已复制到剪贴板"
3. 打开记事本，按 Ctrl+V 粘贴
4. **验证：** 粘贴的内容包含会议名称和会议号

#### 静音/解除静音
1. 点击"静音"按钮
2. **验证：** 按钮状态变为激活（红色背景）
3. **验证：** 图标变为静音图标
4. 再次点击
5. **验证：** 恢复正常状态

#### 开启/关闭视频
1. 点击"关闭视频"按钮
2. **验证：** 按钮状态变为激活
3. **验证：** 本地视频停止
4. 再次点击
5. **验证：** 视频恢复

#### 成员
1. 点击"成员"按钮
2. **验证：** 弹出成员列表模态框
3. **验证：** 显示所有会议成员

#### 聊天
1. 点击"聊天"按钮
2. **验证：** 弹出聊天模态框
3. **验证：** 可以发送和接收消息

#### 录制
1. 点击"录制"按钮
2. **验证：** 显示提示"录制功能暂未实现"
3. **说明：** 这是占位功能，等待后端支持

#### 暂停共享
1. 点击"暂停共享"按钮
2. **验证：** 按钮状态变为激活
3. **验证：** 按钮文字变为"恢复共享"
4. **验证：** 共享的屏幕画面冻结（不再更新）
5. 再次点击"恢复共享"
6. **验证：** 屏幕画面恢复更新

#### 结束共享
1. 点击"结束共享"按钮
2. **验证：** 屏幕共享停止
3. **验证：** 工具栏消失
4. **验证：** 恢复正常会议视图
5. **验证：** 底部控制栏的"共享屏幕"按钮恢复正常状态

### 7. 测试共享音频选项

1. 点击"共享屏幕"按钮
2. 勾选"同时共享电脑声音"
3. 点击"开始共享"
4. 播放一段音频/视频
5. **验证：** 其他参与者可以听到电脑播放的声音（需要另一个客户端测试）

### 8. 测试多用户场景

#### 作为共享者
1. 用户A开始屏幕共享
2. **验证：** 用户A看到工具栏
3. **验证：** 用户A的本地视频显示共享的屏幕

#### 作为观看者
1. 用户B加入会议
2. **验证：** 用户B看到全屏的共享屏幕
3. **验证：** 用户A的摄像头显示在左上角小窗（如果启用了画中画）
4. **验证：** 小窗可以拖动

### 9. 测试边界情况

#### 取消屏幕选择
1. 点击"共享屏幕"
2. 在选项对话框点击"开始共享"
3. 在系统屏幕选择器中点击"取消"
4. **验证：** 不显示错误提示
5. **验证：** 静默返回正常状态

#### 用户停止共享（系统按钮）
1. 开始屏幕共享
2. 点击浏览器/系统的"停止共享"按钮
3. **验证：** 工具栏消失
4. **验证：** 恢复正常会议视图

#### 离开会议时清理
1. 开始屏幕共享
2. 点击"离开会议"
3. **验证：** 所有资源正确清理
4. **验证：** 计时器停止
5. **验证：** 没有内存泄漏

## 预期结果

所有测试应该通过，功能正常工作。如果发现问题：

1. 检查浏览器控制台是否有错误
2. 检查 Vite 开发服务器输出
3. 确保所有依赖已正确安装
4. 尝试重启开发服务器

## 已知限制

1. **录制功能** - 暂未实现，需要后端支持
2. **人像画中画选项** - 选项存在但需要与现有 PIP 逻辑集成
3. **响应式布局** - 工具栏在小屏幕上可能需要优化

## 性能检查

1. 工具栏动画应该流畅（60fps）
2. 计时器更新不应该影响性能
3. 暂停/恢复共享应该立即响应
4. 没有明显的内存泄漏

## 浏览器兼容性

测试以下浏览器：
- ✅ Chrome/Edge（推荐）
- ✅ Firefox
- ⚠️ Safari（可能有限制）

## Electron 特定测试

1. 屏幕选择器显示所有可用屏幕
2. 剪贴板 API 正常工作
3. 窗口焦点处理正确
